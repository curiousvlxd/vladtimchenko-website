## Stack

**Frontend:** Nuxt 3, Vue 3, TypeScript, Tailwind CSS

**Modules:** Nuxt Content (MDX feed), VueUse Motion, @nuxtjs/tailwindcss

**Runtime:** Nitro on Cloudflare Pages. When someone requests `/rss.xml`, `/sitemap.xml` or `/robots.txt`, the same app generates the response. There is no second server or database.

**Content:** Feed posts from `src/data/feed` (file-based, MDX). CV and home copy from JSON.

---

## Integrations

**GitHub:** Stars and repo metadata for the site repo (`/api/site-repo`) and featured projects (`/api/repos`). Uses GitHub REST API with caching.

**Giscus:** Comments on feed posts. Theme and config via `giscus.json` and runtime config. Allowed origins include production and localhost.

**Server routes:** `/rss.xml` (feed), `/sitemap.xml`, `/robots.txt`. All generated at runtime from content and config.

---

## Deploy

**Host:** Cloudflare Pages

**CI:** GitHub Actions on push to `main`. Workflow checks out repo, runs `npm ci` and `npm run build`, then deploys `dist` with Wrangler to Cloudflare Pages (branch `main`). Concurrency cancels in-progress runs.

**Screenshots:** When `src/**`, `public/**`, configs or deps change, the workflow runs Playwright, captures doc screenshots, and pushes them to the `assets` branch in the `screenshots` folder. Images are served from raw GitHub (assets branch).

**Secrets / vars:** `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID`, `CF_PAGES_PROJECT`. Optional: `SITE_URL` for capture base URL, `GITHUB_SITE_REPO` and Giscus env for runtime config.
